TIME-WAIT 是什么，为什么必须等待 2MLS

TIME-WAIT 是一种 TCP 状态。等待 2MLS 可以保证客户端最后一个报文段能够到达服务器，如果未到达，服务器则会超时重传连接释放报文段，使得客户端、服务器都可以正常进入到 CLOSE(关闭) 状态

http 1.0、1.1、2.0的区别

  • 搞懂 HTTP 1.0 /1.1/2.0 协议差别
  • http 1.0 中，一个 tcp 连接只做一件事情
  • http 1.1 中，保持连线的机制，一个 tcp 连接多次重复使用
  • http 2.0 中，多路复用，一个 tcp 连接并行发送多个请求和响应。数据流优先级和服务器推送。
